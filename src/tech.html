Tech stack

For developers and XR/Godot enthusiasts: engine, runtime, audio, and platform details.

At a glance

  • Engine: Godot 4.6
  • Language: GDScript (C#/DotNet assembly present for compatibility)
  • Rendering: Mobile renderer, ETC2/ASTC texture compression
  • XR: OpenXR with Meta (Quest 2/3/Pro), hand tracking, passthrough
  • Export: Android (arm64-v8a), Meta XR plugin, package name RaveSpin

Godot & project setup

RaveSpin is a Godot 4.x project with config/features set to 4.6 and Mobile. The main scene is loaded from the editor; boot splash uses the project’s warm amber colour and a custom image.

Enabled editor plugins:

  • godot-xr-tools — XR interaction, movement, hands, staging (e.g. player height, snap zones)
  • waveform_generator — Waveform export/usage for tracks

Autoloads include XRToolsUserSettings and XRToolsRumbleManager from godot-xr-tools.

OpenXR & Meta Quest

OpenXR is enabled with environment blend mode 2 (e.g. additive/alpha blend for AR-style). Hand tracking is enabled. Meta-specific extensions in use include passthrough; the project also references Meta environment depth and application space warp in shader globals (for future or optional use).

Export preset (Android) enables the Meta XR plugin with:

  • Quest 2, Quest 3, Quest Pro support
  • Passthrough, hand tracking, render model
  • Overlay keyboard

HTC passthrough is referenced in the codebase as well. The Arena scene initialises the XR interface, disables vsync, enables passthrough when supported, and recenters the view on pose recenter.

Audio architecture

Audio is driven by a custom BUS_MANAGER (GDScript) that defines the bus layout and effect slots. Buses include Master, Microphone Input, Channel 1/2 FX and Input (and placeholder Channel 3/4 Input).

Per-channel chain

  • Trim (Slot 0) — AudioEffectAmplify
  • 3-band EQ (Slot 1) — AudioEffectEQ (6-band), bands mapped to low/mid/high (e.g. 32 Hz, 1 kHz, 10 kHz)
  • Sound colour FX (Slots 2–3) — Lowpass and highpass filters
  • Beat FX (Slot 4 when one-FX-at-a-time) — Single beat effect per channel

Beat FX types (FLX4-style)

One effect active per channel at a time (configurable). Types: Delay, Echo, Reverb, Trans, Flanger, Phaser, Pitch, Crush (lofi distortion), Compressor, Limiter, Band Pass, Panner, Stereo Enhance, Distortion. Implemented with Godot’s AudioEffectDelay, AudioEffectReverb, AudioEffectChorus, AudioEffectPhaser, AudioEffectPitchShift, AudioEffectDistortion, AudioEffectCompressor, AudioEffectLimiter, AudioEffectBandPassFilter, AudioEffectPanner, AudioEffectStereoEnhance.

FX policy

Track FX policy can be: FX disabled, only track 1, only track 2, or both tracks. The LibreBox FX UI (e.g. fx_button.gd) adds/removes beat FX per channel based on this policy and the current “can this track take FX?” state.

Song & metadata

Tracks are represented by a Song resource (Song.gd) with: title, main artist, guest artists, album, genres (from a large EGenre enum), audio file, optional waveform texture, BPM, key/scale (e.g. EMusicKey), origin platform (e.g. local), favourite flag, user sidenote, and optional MusicBrainz ID. Metadata can be auto-filled from WAV tags where available.

Key scripts & scenes

  • LibreBox_Manager.gd — Singleton-style LibreBox: track refs, play/pause, load track into controller, BPM getters, hub refresh.
  • Controller.gd (DJ_Controller) — Audio players, crossfader, EQ/trim/colour/beat FX knobs, tempo, load track into memory.
  • BUS_MANAGER.gd — Static bus layout, beat FX enum and names, add/remove/toggle beat FX, EQ weight calculations.
  • arena.gd — XR interface init, passthrough enable, recenter.
  • LibreBox UI — Main hub (active track cards, DB meter, BPM spinner), track selection, add track, FX selection menu (e.g. fx_button.gd, fx_selection_menu.gd).
  • DDJ-FLX4 — 3D controller scene and textures (e.g. FLX4_T_FLX4_Top - Clearer.png, slider).

Building & running

Export from Godot using the Android preset with Meta XR enabled. Target arm64-v8a for Quest. No custom CI is committed; deployment is via Godot’s export. For development, open the Godot folder as the project root in Godot 4.6.
