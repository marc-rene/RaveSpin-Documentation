Tech stack

For developers and XR/Godot enthusiasts: engine, runtime, audio, and platform details.

At a glance

  • Engine: Godot 4.6
  • Language: GDScript (C#/DotNet assembly present for compatibility)
  • Rendering: Mobile renderer, ETC2/ASTC texture compression
  • XR: OpenXR with Meta (Quest 2/3/Pro), hand tracking, passthrough
  • Export: Android (arm64-v8a), Meta XR plugin, package name RaveSpin

Godot & project setup

RaveSpin is a Godot 4.x project with config/features set to 4.6 and Mobile. Enabled editor plugins: godot-xr-tools (XR interaction, movement, hands, staging) and waveform_generator. Autoloads include XRToolsUserSettings and XRToolsRumbleManager.

OpenXR & Meta Quest

OpenXR enabled with hand tracking. Meta extensions: passthrough, Quest 2/3/Pro support. The Arena scene initialises the XR interface, enables passthrough when supported, and recenters on pose recenter.

Audio architecture

BUS_MANAGER defines the bus layout. Per-channel chain: Trim, 3-band EQ, sound colour (lowpass/highpass), beat FX (one at a time per channel). Beat FX types: Delay, Echo, Reverb, Trans, Flanger, Phaser, Pitch, Crush, Compressor, Limiter, Band Pass, Panner, Stereo Enhance, Distortion. FX policy: per-track or both tracks from LibreBox UI.

Song & metadata

Song resource: title, artist(s), album, genres, audio file, waveform, BPM, key/scale, origin platform, favourite, user note, optional MusicBrainz ID.

Key scripts & scenes

  • LibreBox_Manager.gd — Track refs, play/pause, load track, BPM getters.
  • Controller.gd (DJ_Controller) — Audio players, crossfader, EQ/trim/FX knobs, tempo.
  • BUS_MANAGER.gd — Bus layout, beat FX add/remove/toggle.
  • arena.gd — XR init, passthrough, recenter.
  • LibreBox UI — Hub, track selection, add track, FX selection.
  • DDJ-FLX4 — 3D controller scene.
